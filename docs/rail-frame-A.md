# Рамка вокруг рельс — вариант A (graphics_variation)

Тонкая рамка по внешнему контуру сети, на той же машинке масок, что и `rail-art`.
Рамка рисуется на сторонах тайла, **где соседа-рельса нет** (инверсия `present[]`).

## Сущность

Отдельный слой — SEWO `gofarovich-scl-rail-frame` поверх рельса:

- `pictures` = лист вариаций (см. ниже), `render_layer` ниже каретки;
- невыбираемый, без коллизии, `random_variation_on_create = false`;
- создаётся/обновляется вместе с `rail-art`.

## Рантайм

Рядом с расчётом `eff` в `rails.lua`:

```lua
local empty = 0
if not present.N then empty = empty + 1 end   -- bit0
if not present.E then empty = empty + 2 end   -- bit1
if not present.S then empty = empty + 4 end   -- bit2
if not present.W then empty = empty + 8 end   -- bit3
frame.graphics_variation = empty + 1          -- 1-база; вариация 1 = прозрачная
```

Пересчёт только на build/mine соседа (уже захукано через `rail_update`). Per-tick стоимости нет.

## Ассет — лист 16 кадров

- раскладка **4×4**, ячейка = тайл (64px; 128 для HR) → лист 256×256 (512 HR);
- индекс ячейки = `empty_mask` (0..15), порядок бит `N=1, E=2, S=4, W=8`;
- кадр 0 (маска 0 = тайл внутри сети) — **прозрачный**.

Уникальных рисунков ~6, остальное повороты/зеркала:

| маска | вид | вариантов |
|---|---|---|
| `0000` | прозрачный | 1 |
| `0001` | одна грань | ×4 |
| `0011` | внешний угол (две смежные) | ×4 |
| `0101` | полоса (две противоположные) | ×2 |
| `0111` | три грани | ×4 |
| `1111` | коробка | 1 |

## Ограничение

16 кадров не видят диагонали → **вогнутые (внутренние) углы** дают крошечный пропуск в контуре.
Для тонкой рамки обычно незаметно. Если критично — апгрейд до **47-кадрового blob-набора**
(8-соседняя маска, авторить из атласа 4 угловых квадрантов, запечь в 47 кадров; индексация в Lua усложняется).
