# Бэклог: устойчивость постройки (UX)
> Часть ТЗ Space Cart Logistics — индекс в [../readme.md](../readme.md). Смежное: морф/сущность рельса — [hard-decisions.md](hard-decisions.md); условия/`cond_lists` — [signals-and-gui.md](signals-and-gui.md).

Мелкие, но заметные игроку огрехи размещения/копирования. Делаем перед блэкаутом/M7.
- **B1. ✅ Каретку нельзя ставить не на рельс** — `on_built` отклоняет постройку без рельса И на рельс без путей (`eff_mask == 0` — тайл невидим, ехать некуда); возврат предмета + warning. То же в клоне (`on_entity_cloned`).
- **B2. ✅ Адекватное копирование рельса (закрыто v2.5).** Геометрию чертёж несёт сам (прототип+direction рельса-машины) — блюпринт видим и поворачивается корректно. Теги сущности: `scl_dir/mode/conditions_on/cond_lists/cat_order/read_next`; при постройке из бпринта (`event.tags`) стороны в `cond_lists`/`cat_order` ремапятся по разнице `scl_dir` ↔ direction построенной сущности, `manual_mask` берётся из самой сущности. Прямой клон (`on_entity_cloned`, editor) обрабатывается: настройки копируются из узла тайла-источника (клон не поворачивает → ремап сторон не нужен); занятый тайл / каретка без рельса — молчаливый снос клона (игрока-виновника нет).
- **B3. ✅ Один рельс на тайл** — постройка на занятый тайл отклоняется (тот же откат, что B1).
- **B4. ✅ Копирование настроек рельса** (shift+ПКМ пример → shift+ЛКМ вставка): переносится **весь пользовательский ввод** — режим (auto→auto, manual→manual), **галочки путей** (`manual_mask`) и **условия** (`conditions_on`/`cond_lists`/`cat_order`; поле `read_next` упразднено v0.12). НЕ переносится только вычисленное состояние (`eff_mask`/морф) — цель пересчитывает своё (в auto — из СВОИХ соседей). Стороны мировые → ремап не нужен. **Два пути доставки** (перенос идемпотентен): нативный `on_entity_settings_pasted` (одинаковые прототипы; для разных `additional_pastable_entities` обязан слать событие, но в 2.0.76 не шлёт — плюс нативный буфер держит ссылку на сущность и умирает при морфе) и свои linked-инпуты `copy/paste-entity-settings` с источником-ключом тайла (`storage.copy_rail`) — работают между любыми из 22 прототипов и переживают морф; копирование не-рельса сбрасывает источник.
