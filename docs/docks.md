# Доки (M7)
> Часть ТЗ Space Cart Logistics — индекс в [../readme.md](../readme.md). Смежное: качество/инвентарь каретки — [hard-decisions.md](hard-decisions.md); read-next и композит-предикат сравнений — [signals-and-gui.md](signals-and-gui.md); разворот/`reverse_cart`, тай-брейк N>E>S>W — [movement.md](movement.md).

**Док** (loader/погрузчик) — сущность **1×1 с направлением** (4 стороны, как манипулятор). База — **клон `constant-combinator`** (`gofarovich-scl-dock`), по тем же причинам, что у рельса ([hard-decisions.md](hard-decisions.md)): нативные direction (без хаков/стрелок) и провода (источники **R/G** условий захвата), direction в блюпринтах, не крафт-машина. Базовый арт — `integration_patch` (Sprite4Way); состояние (рука 7.1–7.7 / disabled) — **арм-оверлей** `gofarovich-scl-dock-arm`: runtime-сущность `simple-entity-with-owner` с `graphics_variation` (паттерн каретки), невыбираемая и не-блюпринтная — чертёж несёт только сам док. Нативное окно комбинатора подавлено. В отличие от рельса, **поворот R разрешён** (dir перечитывается каждый тик). Направление = с какого соседнего тайла док **выдёргивает** каретку (роборука в 3D-анимации). Пока только **твёрдые** доки; жидкие (для жидких кареток/хранилищ) — позже, отдельным контентом. Твёрдую каретку берёт твёрдый док, жидкую — жидкий.

**Док = хранилище, но только с кареткой.** Пойманная каретка становится инвентарём дока — из него берут манипуляторы/боты (как из сундука). Без каретки док **пуст** (не «бесплатный контейнер»: в космосе хранилищ мало по дизайну — склад это каретки в движении). Заполнение/опустошение — ванильным арсеналом (манипуляторы, фильтры, логистика), собственной логики «что грузить» у дока нет. **Скелет пока этого не даёт:** груз каретки — скриптовый LuaInventory, манипуляторам он не виден; в скелете каретка на доке остаётся выбираемой сущностью, груз открывается по E как обычно. Мост к ванильному арсеналу (proxy на инвентарь каретки нельзя — `proxy-container` целится только в инвентарь-ИНДЕКС сущности, у simple-entity его нет; вероятно runtime-сундук-компаньон с post-tick синком) — **отдельная задача после условий**.

**Геометрия захвата:**
- Берём только с **прямого участка** рельса на целевом тайле (`|` или `—`). С углов/поворотов — **нет**.
- Прямой участок бывает двух ориентаций относительно оси взгляда дока:
  - **сквозной** (рельс перпендикулярен взгляду): каретки текут мимо дока; опущенная едет **туда же, куда ехала** (см. «направление» ниже).
  ```
  → ═ ═ ═ →     рельс E–W, каретки текут мимо
      D         док смотрит N; ловит проезжающую, опускает — едет вправо
  ```
  - **лобовой** (рельс идёт вдоль взгляда, В док): каретка приходит навстречу доку; опущенная едет с **обратным** направлением, чтобы уехать — естественный терминал/тупик:
  ```
      ↓         рельс N–S идёт в док
      ═
      D         док смотрит N; каретка приезжает сверху, опускается — уезжает вверх
  ```
- Док, смотрящий на тайл **без прямого пути** по нужной оси (или где рельса вообще нет) → **выключен** (disabled-спрайт, ловить нечего).
- Несколько доков на один рельс-тайл распределяются по **приоритету сторон света** — тот же тай-брейк, что у голов составов: **N > E > S > W** (см. [movement.md](movement.md)).

**Направление пойманной каретки** хранится в доке: каретка помнит, куда ехала, и после опускания продолжает тот же курс (в лобовом случае — инвертированный, чтобы уехать от дока).

**Два вида условий (как у поездов): «когда брать» и «когда опускать».** Два **независимых** редактора в двух соседних окнах:
- **Условие захвата** («когда брать каретку») — гейтит анимацию 7.2/взятие; описано ниже.
- **Условие отпускания** («когда опускать каретку») — независимый набор в соседнем окне, концептуально как условия отправления у поездов. Тот же композит И/ИЛИ, источник **Cart** здесь = **пойманная** каретка (контейнер, уже принадлежащий доку). Примеры: «каретка опустела», «внутри ≥1 метеорита» (сравнение количества предмета с целевым). **Подробности — позже**; текущие тесты сосредоточены на маршрутизации и **подборе** кареток (захват), отпускание пока по дефолту.

**Условие захвата = композиция сравнений по И/ИЛИ** (модель сравнивающего комбинатора; **только у доков** — у рельсовых условий маршрута И/ИЛИ нет). Одно **сравнение** = `[сигнал] [оператор] [сигнал | константа]` (`COMPARATORS = {"<", ">", "=", "≥", "≤", "≠"}`). Сравнений может быть несколько, между соседними — связка **И**/**ИЛИ**; семантика — **ДНФ** (И крепче ИЛИ: `(a И b) ИЛИ c ИЛИ (d И e)`), полна для любой булевой комбинации. Пустое сравнение = невыполнено (false) (как у рельсовых условий). Каретка **«валидна»** (запускает 7.2) ⇔ предикат захвата для неё истинен. Примеры: «брать только пустую», «с нужным ресурсом», «с одним из двух».
- **Источник сигнала — три галочки R / G / Cart** у каждого операнда-**сигнала**:
  - **R**/**G** — red/green провод дока (док нативно wire-connectable);
  - **Cart** — содержимое каретки как сигналы: в **захвате** — той **подъезжающей**, за которой следит рука (см. анимацию); в **отпускании** — **пойманной** (контейнер дока).
  - Значение = **сумма** сигнала по выбранным источникам. Галочка неактивна, если провод не подключён (R/G) или операнд — константа. **Ни одной галочки → читаем 0** (ничего).
- **GUI** (как decider combinator): строки-сравнения с **отступом слева**; связки И/ИЛИ в левом гуттере **между каждой соседней парой** строк (только при **2+**), клик переключает И↔ИЛИ; **ИЛИ «главнее», левее** (отступ справа), **И правее** (отступ слева); слева **скобки** охватывают И-группы. Внизу — кнопка **«+ Add condition»** (добавляет строку-сравнение, по умолчанию И к предыдущей). Раскладка — чисто для считывания ДНФ; на семантику влияет только выбор И/ИЛИ.
- **Правый операнд**: константа или сигнал (при константе R/G/Cart этого операнда гаснут). Сокращение числа/поп-ап ввода — как у рельсового операнда-константы (см. [signals-and-gui.md](signals-and-gui.md)).

**Анимация захвата.** Рука дока — одна ось «убрана ↔ схватить»; её движение **следит за подъезжающей кареткой**, а не крутится по таймеру. Пока по целевому тайлу **к центру продолжает ехать валидная** каретка — рука идёт вперёд (7.2); нет такой каретки (уехала, встала, игрок развернул, до центра не дойдёт) — рука идёт назад (7.7). Прерывания резюмятся **с текущего кадра** (не с начала) — не теряем время на дёрганье. Хватает рука **только ту** каретку, что реально дошла до центра валидной. «Валидная» = проходящая условие захвата.
- **7.1 Ожидание** — один кадр, просто неактивный пустой док.
- **7.2 Подготовка** — валидная каретка **только** заезжает на тайл и занимает собой пока **только первую внутреннюю клетку**; рука дока начинает движение к ловле. Идёт вперёд, **пока эта каретка продолжает ехать к центру**. Полная длительность = **половина** времени проезда каретки по тайлу (16 тиков на прямом = 32 клетки) → рука доходит до хвата ровно когда каретка — до центра.
- **7.3 Взятие** — валидная каретка дошла до **центра** тайла (рука полностью вышла): рука выхватывает каретку, док переходит в загруженное положение.
- **7.4 Загружен** — статичный кадр загруженного кареткой дока (док = инвентарь).
- **7.5 Опускание** — **реверс 7.3**: док опускает каретку над рельсом.
- **7.6 Ожидание места** — док держит последний кадр 7.5, пока рельс по центру не станет пуст, тогда каретка встаёт на него.
- **7.7 Возврат в ожидание** — реверс 7.2, автопроигрывание, **когда валидной каретки к центру нет**. Прерывается 7.2 в любой момент — с того кадра, на котором оборвался.

**Крайние случаи** (почему рука *следит*, а не таймерит от въезда): перед доком могут встретиться две каретки навстречу; дойти до центра валидной может помешать каретка с другого пути (перекрёсток); игрок может развернуть/перенаправить каретку на ходу. Во всех — рука идёт вперёд лишь пока конкретная валидная каретка приближается к центру, иначе откатывается; хват — строго по факту «валидная в центре». Так анимация всегда вовремя включается (на въезде) и поднимается **только правильная** каретка.

## Скелет (шаг 3 M7 — реализовано, `scripts/docks.lua`)
Без условий: любая каретка «валидна», отпускание — командой `/scl-dock-release` (условия захвата/отпускания — шаги 4–5). Состояние: `storage.docks[key] = { x, y, entity, visual, dir, tkey, enabled, state, arm, watch, watch_i, held, heading, last_released }`.

- **Синхрон руки по построению.** «Каретка заезжает на тайл» = её **центр** (спрайт) пересёк край: центр = `cells[head−16]`, при прямом сегменте это `seg[i−16]` ⇒ центр на тайле ⟺ `cursor.i ∈ 17..32`, клетка центра `c = i−16 ∈ 1..16`, центр тайла = `seg[16]` (t=0.5). Кадр руки = `c` (0..16=REACH): въехал центр (кадр 1) → рука стартует, центр в центре (c=16) → рука дошла — ровно **половина проезда тайла = 16 тиков**, как в 7.2. Рука шагает ±1 кадр/тик к `c` (прерывание резюмится с текущего кадра), «каретка встала не в центре» = кадр не растёт (по `watch_i`) → откат.
- **Хват** (7.3, в скелете мгновенный): `c == 16` **и** рука дошла (`arm == 16`). Встала В центре — тоже хват (лобовой тупик стопорит каретку ровно в центре — `next_head` за тайлом нет). Захват = `C.cart_detach` (вырез из состава с сохранением записи и груза) + телепорт сущности на док; курс (`cursor.exit`) запоминается в `heading`.
- **Опускание** (7.5/7.6): `exit = heading`, лобовой (`heading == OPP(dir)`) — инверсия, чтобы уехала; ждём, пока прямой путь этого курса существует **и всё окно 32 клеток свободно** (не только центр: окно = клетки, которые займёт тело; иначе наложение тел) → `C.cart_attach(..., require_free)`. Затем откат руки (7.7). Гард `last_released`: только что опущенная не ловится, пока её голова на целевом тайле (иначе застрявшая в центре каретка зациклила бы хват-отпуск).
- **Владелец тайла**: из свободных включённых доков на один целевой тайл ловит один — приоритет стороны дока (относительно тайла = `OPP(dir)`) **N > E > S > W**, как у голов составов. Кандидат-каретка: голова на тайле, сегмент прямой, `i > 16`; из нескольких — ближняя к центру (max `i`, тай-брейк меньший `unit_number`). Один проход по кареткам за тик, обход доков по сортированным ключам (детерминизм).
- **Снос**: дока — пойманная каретка остаётся стоять на месте дока (легальное «каретка без рельса»), добывается отдельно; каретки на доке — док замечает лениво (тик) и откатывает руку. Блэкаут цели — каретка гибнет штатно, док откатывается. Rebuild мода — `Docks.rebuild()` (перечитывает сущности, релинкует пойманную по `unit_number`, пересоздаёт арм-оверлеи).
- Отладка: `/scl-dock-info` (дамп состояний), `/scl-dock-release`.
