# Space Cart Logistics — ТЗ (v2.6)

> **База рельса — 22 прототипа `constant-combinator`** (одна сущность на тайл; маска =
> класс × `direction`, контракт `scripts/railmask.lua`; арт в `integration_patch`).
> Двухсущностная схема примари+арт (v2.3) и `assembling-machine` (v2.5.x) отвергнуты —
> детали и обоснование в [docs/hard-decisions.md](docs/hard-decisions.md). Флип чертежей
> не поддержан (у комбинаторов нет mirroring-бита).
>
> **Модель сигналов — направленная (v2.4):** геометрия пути = галочки/соседи; куда
> поедет каретка — упорядоченный список условий «(вход → выход): предикат» (12 видов =
> 4 входа × 3 поворота). Старая модель «6 условий по соединению, гейтящих геометрию» отменена.

## Цель
Factorio-мод (2.0 / Space Age): тайл-размерные рельсы и каретки, едущие по ним как «смесь конвейера и поезда». **Это не пасфайнд**: у кареток нет маршрута A→B, они просто текут. «Программа» движения живёт в состоянии соединений тайлов (задаётся сигналами), интеллект в сети, не в каретке. Вся механика на Lua; родные trains/rails/cars/LuaRendering **не используются**.

## Стек
Factorio 2.0 API (Lua 5.2). `storage`, не `global`. Файлы: `info.json`, `data.lua`, `control.lua`, `graphics/`.

## Нейминг
- Mod: **Space Cart Logistics**. Internal name (`info.json` `name`, путь `__space-cart-logistics__`): `space-cart-logistics`. Автор: `gofarovich`.
- **Префикс всех именованных регистраций — `gofarovich-scl-`**: прототипы, item'ы, entity, рецепты, custom-input'ы, remote-интерфейсы, шорткаты и т.п. Напр.: `gofarovich-scl-rail`, `gofarovich-scl-cart`.
- Lua-идентификаторы (где дефис недопустим — ключи/функции/локали) — под тем же неймспейсом, но через подчёркивание: `gofarovich_scl_`.

## Этапы
1. **База (готово):** прототипы рельса/каретки (off-grid, graphics_variation, teleport), граф рельс, одна каретка едет по связному пути.
2. **Клеточная модель (готово):** path→клетки (32/25), каретка = окно 32 клетки, 1 клетка/тик, поворот по касательной. Одна каретка по правилу `прямо→право→лево→стоп`. Проверка: нет сжатия на повороте.
3. **Один состав (готово):** snake-deque, push/pop, движение от головы, маршрут по снапшоту, join/split на линии прямая+поворот. Голова встала → все встали.
4. **Кольцо (готово):** детект цикла + атомарный проворот; полное кольцо без пузырька крутится.
5. **Перекрёстки + арбитраж (готово):** клеточные конфликты, ожидание всего состава, ПДД → N>E>S>W. Две пересекающиеся струи.
6. **Сигналы (готово):** GUI тайла, **направленные условия** (12 видов «вход→выход», список по входу, first-match-wins, фолбэк на дефолт), чтение сети примари-комбинатором (по-тиковый кэш), геометрия по галочкам → `graphics_variation` (64 вариации), блэкаут-взрыв, **read-next (6h)** — груз входящей каретки мерджится в условия тайла-цели и выводится в его комбинатор. См. [docs/signals-and-gui.md](docs/signals-and-gui.md).
7. **Груз и станции (M7, в работе):** модель груза каретки ✅ — скриптовый инвентарь, нативное окно по E (число слотов = качество каретки, 1–5, порядок в цепочке качеств, `G.slots_by_quality`; см. [docs/hard-decisions.md](docs/hard-decisions.md)); read-next ✅ (6h); **доки** (см. [docs/docks.md](docs/docks.md)). **Позже:** кросс-составной gridlock.

UX-правки вне этапов (selection_priority, рельс под кареткой, разворот по R) — в [docs/movement.md](docs/movement.md).
